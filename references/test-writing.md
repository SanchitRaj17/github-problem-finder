# Test Writing Tips

- No surprise tests: everything tested must be part of the requirements.
- Tests are adequate: comprehensive coverage and sufficient depth.
- Tests are not brittle: avoid over-constraint and implementation-specific expectations.
- Use repository-conventional method names and patterns.
- A correct implementation should pass all tests without hacks.

## Questions to ask

- Are the tests sufficient to validate the behavior described in the problem?
- Are there extra tests that do not make sense?
- Are there tests that can be removed because they are not relevant to the problem statement?
- Are any important edge cases missing?
- Are any tests brittle?
